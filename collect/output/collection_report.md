## Reproducibility

- **Offline mode** is fully deterministic: it reads `fixtures/*.json` and applies
  the same normalization and merge rules every run.
- **Live mode** (`node run.mjs --mode live`) queries the unofficial Google
  Autocomplete endpoint (free) and, if `SERPAPI_KEY` is set, People Also Ask via
  SERP API. It requires a network on which those hosts are reachable.
- Every canonical question records its `variantIds`, so each merge can be traced
  back to the exact raw records that produced it.

## Notes & limitations

- The country signal is the **question surface served to each locale**, set via
  `gl`/`hl` — not per-person geodata. Age and gender are out of scope.
- Frequency is a **relative** indicator; no source provides true volume.
- Near-duplicate merging here is lexical (content-token signature + Jaccard) for
  full transparency; production may substitute embeddings.
- Korean tokenization is lightweight (particle stripping); it is sufficient for
  Stage 100 but not linguistically complete.
